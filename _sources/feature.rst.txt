Dynamic CPU Pinning Optimization
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
Susanooは受け取るトラフィックに対してCPU割り当てを動的に最適化します。
ポートの再設定は以下の手順で行われる.

 1. Setting parameters
 2. Stop port
 3. Configuration port
 4. Start port

この手順のみ
実効しながらユーザが明示的に指定
パフォーマンスからSusanooが自らシステム再構成


Cpu Pinning Pattern
^^^^^^^^^^^^^^^^^^^
分割の粒度
 - 処理の規模が大きいほどよい影響がありそう
 - 規模が大きくないのを無理に分割するとオーバーヘッドになりそう

→ホットスポットの粒度をあげればなんか気持ちよくなれそう

処理の分割
 - ポート分散→別ポートが安定
 - 機能分散　→別の機能が安定
 - Flow分散  →別Flowが安定

選択のポイント
 - どれに負担がかかるのか
 - どれを安定させたいのか
